# Webkassa API — заметки из документации

**Версия:** ИНТЕГРАТОРЫ_v4-2.0.3
**Базовый URL (тест):** https://devkkm.webkassa.kz
**Дата обновления документации:** 30.03.2026

---

## История изменений (последние)

- **30.03.2026** — В методе `/v4/cashbox/client-info` добавлена информация о контактах ЦТО и о лицензии.
- **16.03.2026** — Расширено описание параметра `RoundType` метода `/api/check`. Обновлено описание заголовка `Accept-Language` в `/v4/Ticket/PrintFormat`.
- **05.02.2026** — Улучшена структура документации, добавлен раздел «Список кодов ошибок», метод `/api/check` дополнен примерами возвратов для протоколов 2.0.2 и 2.0.3.

## Известные методы (из заметок)

- `POST /api/authorize` — авторизация (логин/пароль → токен)
- `POST /api/check` — фискализация чека (продажа и возврат)
- `GET  /v4/cashbox/client-info` — информация о кассе (контакты ЦТО, лицензия)
- `GET  /v4/Ticket/PrintFormat` — формат для печати чека
- (есть Z-отчёт по смене)

## Коды ошибок (выдержка)

| Код | Описание | Что делать |
|---|---|---|
| **18** | Превышен лимит автономного режима (72 часа без связи с ОФД) | Проверить оплату ОФД, наладить интернет |
| **505** | Сервис недоступен по этому домену | Использовать альтернативный домен из `Response.HttpHeaders['AlternativeDomainNames']` |
| **1014** | Смена ещё открыта, Z-отчёт отсутствует | Перепроверить даты в запросе |

## Failover: альтернативные домены

Если получили **505**, сервер возвращает в HTTP-заголовке `AlternativeDomainNames`
список запасных доменов через запятую. Логика:

1. Получили 505 → парсим заголовок
2. Берём первый домен → пробуем повторить
3. Если снова ошибка → следующий
4. И так далее

**Реализовать в `webkassa_client.js`** — это обязательная часть для надёжной работы.
