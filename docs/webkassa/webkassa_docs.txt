Обзор сервиса Webkassa

Данный документ содержит описание сервиса 
Webkassa
, который используется для фискализации кассовых операций и передачи данных об этих операциях Оператору фискальных данных (ОФД).

Сервис позволяет работать с кассовыми операциями и кассовыми документами, формируемыми ОФД и/или Webkassa.

Авторизация
Авторизация системы выполняется посредством передачи 
API-KEY
.

Описание API-KEY

Для взаимодействия с API Webkassa выдается специальный ключ 
API-KEY
 — это строка специального формата, идентифицирующая систему, обращающуюся к API.

Типы API-ключей

Для каждой системы создаются два 
API-KEY
:

для тестирования  

для использования в продуктивной среде

Каждый запрос к API Webkassa должен содержать 
API-KEY
 системы, которая обращается к API Webkassa.

Получение API-KEY

Получить 
API-KEY
 для тестовой и продуктивной среды можно через каналы связи с технической поддержкой Webkassa.

AUTHToken

Для удобства тестирования в методе 
/api/Authorize
 используется скрипт, который сохраняет авторизационный токен в переменную 
AUTHToken
.
Данная переменная автоматически используется во всех методах коллекции.

POST
/api/v4/Authorize
https://devkkm.webkassa.kz/api/v4/Authorize

Метод выполняет авторизацию системы в API Webkassa по 
API-KEY
 и инициализирует сессию для последующих запросов.
