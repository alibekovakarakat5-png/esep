# Outreach — платформы (такси, доставка, курьерки)

Цель: за неделю начать 5-10 разговоров. Закрыть за неделю не выйдет — это
pipeline. Но первый созвон/демо реально получить.

---

## ICP — кто реально наш клиент

Подходит платформа, которая **сама выплачивает деньги многим
курьерам/водителям-физлицам** (транзакционная модель). По НК 2026 она —
налоговый агент.

НЕ подходит: P2P-модель (заказчик платит исполнителю напрямую) — удержать
налог нельзя.

**Важно:** гиганты (inDrive, Яндекс, Kaspi/Chocofood) уже построили своё или
в пилоте КГД. Наш рынок — **средние и региональные игроки**, кому строить
дорого и кто ещё не закрыл compliance.

## Как найти ЛПР (контакты не выдумываем — ищем)

1. Сайт компании → раздел «Контакты», «О нас», иногда прямой email/телефон.
2. 2GIS / Google Maps — телефон и адрес офиса.
3. LinkedIn — искать «COO», «операционный директор», «финансовый директор»,
   «head of operations» этой компании.
4. Instagram-аккаунт компании → директ (для небольших — рабочий канал в KZ).
5. Общий `info@` / WhatsApp на сайте — просить переключить на того, кто
   отвечает за работу с курьерами и налоги.

Цель — выйти на операционного или финансового руководителя, не на поддержку.

## Холодное сообщение (WhatsApp / Instagram-директ)

Коротко, про их мир, один лёгкий вопрос. НЕ пресс-релиз.

```
Здравствуйте! Вопрос по новому НК 2026.

С 2026 платформа, которая платит курьерам, — налоговый агент:
обязана удерживать соцплатежи, фискализировать выплаты,
следить за лимитом 300 МРП на человека.

Мы (Esep) сделали API, который закрывает это одним вызовом —
все договоры с КГД и ОФД на нашей стороне.

Как вы сейчас решаете эту задачу? Если интересно — покажу
за 10 минут, как это работает.
```

Вариативные куски:
- «платит курьерам» / «выплачивает водителям» / «рассчитывается с исполнителями»
- «Как вы сейчас решаете эту задачу?» / «У вас это уже закрыто или в процессе?»

## Холодное письмо (email — для Тир 3 и крупнее)

Тема: `налоговый агент по НК 2026`

```
Здравствуйте!

С 2026 года платформа, выплачивающая курьерам, по НК РК признаётся
налоговым агентом — удержание соцплатежей, фискализация каждой
выплаты, контроль лимита 300 МРП на курьера.

Esep — API, который закрывает это одним запросом на выплату.
Договоры с КГД и операторами фискальных данных — на нашей стороне,
вы интегрируетесь только с нами.

Подскажите, кто у вас отвечает за это направление? Покажу демо
за 10 минут — без обязательств.

Каракат, Esep
business.esepkz.com · +7 705 991 47 89
```

## Follow-up (нет ответа 4-5 дней)

Новый угол — конкретная цифра/риск, не «напоминаю».

```
Добавлю по сути: главный риск — лимит 300 МРП. Если курьер его
превысил, он незаконно остаётся самозанятым, и вопрос прилетает
платформе. Наш API блокирует выплату до превышения автоматически.

Показать на демо? 10 минут.
```

Breakup (последнее касание):
```
Не буду надоедать — закрываю тему. Когда дойдут руки до compliance
по курьерам, напишите: business.esepkz.com. Удачи!
```
